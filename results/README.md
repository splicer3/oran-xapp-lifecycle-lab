# Results

This directory contains small, sanitized result samples only. The samples document artifact formats and provide lightweight inputs for local script checks; they are not general performance benchmarks.

## What Is Committed

| Path | Format | Purpose |
| --- | --- | --- |
| `results/sample/ab-testing/csv/ABTesting_kpimonV1.csv` | CSV | Small dashboard-style throughput sample for one `kpimon-go` version. |
| `results/sample/ab-testing/csv/ABTesting_kpimonV2.csv` | CSV | Small dashboard-style throughput sample with two `kpimon-go` versions. |
| `results/sample/ab-testing/conclusions/statistical_conclusion_2026-03-08T14:25:11Z.md` | Markdown | Example statistical conclusion generated from thesis-era Time-Based Switching artifacts. |
| `results/sample/rate-limit/csv/rate_limit_demo_200.csv` | CSV | Optional HTTP rate-limit demo success-rate sample. |
| `results/sample/rate-limit/csv/rate_limit_demo_429.csv` | CSV | Optional HTTP rate-limit demo rate-limited-response sample. |
| `results/sample/rate-limit/csv/rate_limit_demo_markers.csv` | CSV | Marker timestamps for the optional rate-limit demo plot. |

The A/B CSV samples contain human-readable rate strings from exported output. They are useful for showing file shape and smoke-testing script paths. For numeric analysis, prefer Prometheus JSON snapshots or numeric pivot CSV files generated by `ansible/istio-ab-testing/scripts/plot_ab.py`.

## What Is Not Committed

The public repository intentionally excludes:

- raw Prometheus run archives unless reduced to small reviewed samples;
- generated Plotly HTML files;
- dashboard screenshots;
- large logs;
- packet captures (`.pcap`, `.pcapng`);
- kubeconfigs, inventories, tokens, private addresses, or credentials;
- thesis PDFs or personal material.

Generated run outputs stay under ignored `artifacts/` directories during local experiments.

## Regenerating Time-Based Switching Artifacts

Deploy the main RIC lifecycle stack first, then make Prometheus reachable from the controller at the `prometheus_url` configured in `ansible/istio-ab-testing/group_vars/all.yml`.

```bash
cd ansible/istio-ab-testing
cp inventory/hosts.ini.example /tmp/istio-ab-hosts.ini
${EDITOR:-vi} /tmp/istio-ab-hosts.ini
ansible-playbook playbooks/run_demo.yml -i /tmp/istio-ab-hosts.ini
```

That playbook writes ignored local artifacts under `ansible/istio-ab-testing/artifacts/metrics/`:

- `istio_tcp_*.json`: Prometheus `query_range` response for `istio_tcp_received_bytes_total`;
- `flip_events_*.json`: route-switch apply markers;
- `istio_tcp_*.csv`: pivoted version/time table generated by `plot_ab.py`;
- `istio_tcp_*.html`: Plotly chart when Plotly is available.

To run the plotting script against a committed sample CSV:

```bash
cd ansible/istio-ab-testing
python3 scripts/plot_ab.py \
  --in-csv ../../results/sample/ab-testing/csv/ABTesting_kpimonV2.csv \
  --out-csv /tmp/ab-sample.csv
```

`plot_ab.py` input formats:

- Prometheus JSON snapshots from `query_range`, with time/value samples and version labels;
- or a pivot CSV where the first column is time and later columns are version series;
- optional `--flip-events` JSON containing switch markers.

`plot_ab.py` outputs:

- pivot CSV via `--out-csv`;
- Plotly HTML via `--out-html` when `plotly` is installed.

## Regenerating Optional Rate-Limit Demo Artifacts

The rate-limit demo is separate from the RIC lifecycle path. After its K3s/Istio demo environment is running and Prometheus is reachable at the configured URL, run:

```bash
cd ansible/istio-rate-limit-demo
ansible-playbook playbooks/istio/rate_limit_demo.yaml
```

The playbook writes ignored local artifacts under `ansible/istio-rate-limit-demo/artifacts/rate-limit/`:

- `rate_limit_demo_200.csv`: successful HTTP response throughput;
- `rate_limit_demo_429.csv`: rate-limited HTTP response throughput;
- `rate_limit_demo_markers.csv`: enable/effective/disable markers;
- `rate_limit_demo_plot.html`: Plotly chart.

To render the committed sample CSVs:

```bash
cd ansible/istio-rate-limit-demo
python3 scripts/plot_rate_limit.py \
  --success-csv ../../results/sample/rate-limit/csv/rate_limit_demo_200.csv \
  --limited-csv ../../results/sample/rate-limit/csv/rate_limit_demo_429.csv \
  --markers-csv ../../results/sample/rate-limit/csv/rate_limit_demo_markers.csv \
  --output /tmp/rate-limit-sample.html
```

`plot_rate_limit.py` requires `plotly`. Its input CSVs use `time` plus one numeric value column, and the marker CSV uses `time,label`. It writes one HTML plot.

## Statistical Analysis Notebook

The statistical analysis artifact is `experiments/time-based-switching/ab_statistical_validation.ipynb`. It is not part of the Ansible run path. It expects archived Time-Based Switching metrics and flip-event files, then writes:

- `stat_results_<utc>.csv`;
- `routing_and_latency_<utc>.html`;
- `statistical_conclusion_<utc>.md`.

The notebook currently declares historical data/output roots in its first parameter cell. Review those paths before execution in this curated public layout. A standard execution command is:

```bash
jupyter nbconvert --execute --to notebook --inplace experiments/time-based-switching/ab_statistical_validation.ipynb
```

Raw archives used for statistical analysis are not committed here. Keep regenerated analysis outputs out of git unless they are reduced, sanitized, and intentionally added as small samples.
